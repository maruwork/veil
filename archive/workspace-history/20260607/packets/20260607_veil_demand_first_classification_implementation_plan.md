# VEIL 高需要先行判別ルール 実装計画

Status: Draft
Date: 2026-06-07

## 1. ゴール

VEIL の正本文書と capture 手順に、高需要語から段階導入する判別・採用ルールを整合して反映する。

## 2. 道のり

1. 現行文書の不足点を確定する
2. ルール本文を README に入れる
3. `AGENTS.md` に delegated AI 向け要点を入れる
4. 設計書へ判別順・採用優先度・保留条件を書く
5. `docs/manual.html` に UI 利用者向けの段階導入説明を入れる
6. capture 手順へ同じ順序を入れる
7. 各面の表現ずれを見直す

## 3. チェックポイント

### 3.1 原則固定

- 高需要先行
- 判別先行
- 保留明示

### 3.2 面ごとの責務分担

- README: 運用原則
- 設計書: 判別と採用の設計
- skill: 実行順序

### 3.3 整合確認

- 同じ語を別名で呼ばない
- 採用順序が 3 面で一致する
- 保留条件が 3 面で一致する

## 4. タスク

1. `README.md` に段階導入の運用原則を追加する
2. `AGENTS.md` に高需要先行の運用原則を追加する
3. `docs/veil-design.md` に判別順・採用優先度・保留条件を追加する
4. `docs/manual.html` に自動推定と採用順の関係を追加する
5. `skills/codex/veil-capture/SKILL.md` を更新する
6. `skills/claude-code/veil-capture.md` を更新する
7. 差分を見直し、必要なら表現を寄せる

## 5. 検査

- UTF-8 読みで対象ファイルを再読し、同じ原則が書かれているか確認する
- `rg` で `高需要`, `保留`, `判別`, `識別子`, `固有名`, `境界が曖昧` を引いて整合を確認する

## 6. 完了の定義

- user が望んだ「一気に判別→変換ではなく、需要が高いものから徐々に」という運用が正本に入っている
- 次の delegated AI が README / 設計書 / skill のどこを読んでも同じ運用で動ける
