# VEIL seed 語彙整合 要件

Status: Draft
Date: 2026-06-07

## 1. 目的

`app.py` の初期 seed 語彙を、高需要先行と慎重側の判別ルールに合わせて整理する。

## 2. 背景

- いまの `SEEDS` は広く、単語単体や曖昧語まで多く含む
- 新しい運用では、一気に広げず、高需要で意味が安定した語から段階導入する
- UI の自動推定は単語単体を慎重側へ倒すように変わったが、seed は旧方針のまま残っている

## 3. 今回の範囲

- `app.py` の `SEEDS` を見直す
- seed 方針を README / 設計書 / manual に必要最小限で反映する

## 4. 今回の範囲外

- 既存 `vocab.db` の自動移行や掃除
- `~/.veil/rules/` の整理
- UI 機能追加

## 5. 機能要件

### 5.1 seed 方針

- seed は「新規 DB に最初から入れてよい高需要・高確度語」に絞る
- 単語単体で意味が広く揺れる語は seed から外すか慎重側へ寄せる
- `project 固有語` の seed は原則避ける

### 5.2 seed 候補の方向

- 維持:
  - `close`
  - `close record`
  - `blocker`
  - `stale blocker`
  - `verification`
  - `summary`
  - そのほか意味が安定していて説明語として使いやすいもの
- 見直し対象:
  - `active`
  - `live`
  - `current`
  - `workflow`
  - `canon`
  - `reroute`
  - 単語単体で用途が広いもの

### 5.3 文書整合

- helper DB の seed は補助であり、正本ではないことを明確に保つ
- seed は少数の出発点であり、完全辞書ではないことを書く

## 6. 非機能要件

- 新規 DB の初期状態を分かりやすくする
- 日本語訳が不自然な seed を減らす

## 7. 完了条件

- `SEEDS` が高需要・高確度寄りの小さな集合へ整理される
- 文書側に helper seed の位置づけが反映される
