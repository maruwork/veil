# VEIL UI カテゴリ自動推定整合 要件

Status: Draft
Date: 2026-06-07

## 1. 目的

UI のカテゴリ自動推定を、`veil-normalize.py` の判別補助とできるだけ同じ方向へ寄せる。

## 2. 背景

- `veil-normalize.py` には、識別子候補 / 固有名候補 / 説明語候補 / 境界が曖昧な候補の軽い判別補助が入った
- しかし UI の `inferCat()` はまだ簡易な正規表現だけで、判別補助と説明がずれる
- user が UI で登録する時、補助スクリプトと UI の推定が逆を向くと運用がぶれる

## 3. 今回の範囲

- `ui/js/convert.js` の `inferCat()` を見直す
- 必要なら `ui/js/ui.js` の利用箇所を追随する
- `docs/manual.html` のカテゴリ自動推定説明を実装に合わせる
- README / 設計書へ必要最小限の補足を入れる

## 4. 今回の範囲外

- `app.py` の DB schema 変更
- `project 固有語` の自動判定
- `veil-normalize.py` の追加拡張
- `veil-lint.py` の変更

## 5. 機能要件

### 5.1 自動推定の整合

UI の自動推定は少なくとも次に寄せること。

- `key=value`, パス, 拡張子付き名, ALL_CAPS, チケット ID は `2`
- 大文字小文字混在語は `5`
- 小文字中心の空白区切りフレーズは `1`
- 小文字中心だがハイフン/アンダースコアを含む語は `7`
- 判定が揺れる単語は `7`

### 5.2 慎重側

- `project 固有語 (6)` は自動では付けない
- 迷う語は `7` に寄せる

### 5.3 説明整合

- manual の自動推定表が実装と一致する
- 「自動推定は入口であり最終決定ではない」が残る

## 6. 非機能要件

- 既存の変換ロジック自体は変えない
- 既存 UI フローを壊さない
- 日本語説明を優先する

## 7. 完了条件

- UI の `inferCat()` が `veil-normalize.py` の判別補助と大筋で一致する
- `docs/manual.html` の表と note が実装と一致する
