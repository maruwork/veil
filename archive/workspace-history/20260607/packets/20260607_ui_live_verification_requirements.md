# UI 実機確認 要件

## 1. 背景

- 棚卸し導線は code-level check と文書整合までは進んでいる。
- ただし、ブラウザ実機で end-to-end に触った確認はまだ不足している。

## 2. 今回の対象

- `app.py` を localhost で起動する。
- ブラウザ実機で次を確認する。
  - 語彙一覧の監査バッジ、件数サマリ、status 絞り込み
  - `棚卸し優先` sort
  - `要見直し` button によるフォーム送出
  - `現在の要見直し` の表示、保存後の解除、別語編集移行時の解除
  - `削除候補` 一括削除 confirm と preview

## 3. 対象外

- UI 全面の視覚調整
- モバイル表示検証
- 新機能追加

## 4. 完了条件

- 上記対象について、実機で再現手順と結果を残す。
- 不整合があれば findings として切り出す。
- findings が小さければ同セッションで修正候補まで整理する。
