# VEIL UI カテゴリ自動推定整合 基本設計

Status: Draft
Date: 2026-06-07

## 1. 目的

UI 登録時のカテゴリ自動推定を、現在の VEIL 判別補助と同じ方向へ寄せる。

## 2. 境界

- 対象:
  - `ui/js/convert.js`
  - `docs/manual.html`
  - 必要なら `README.md`
  - 必要なら `docs/veil-design.md`
- 非対象:
  - runtime DB
  - `project 固有語` 自動判定
  - 変換エンジン本体

## 3. 主要設計

### 3.1 UI 推定は cat に落とす

`veil-normalize.py` は説明ラベルを返すが、UI は cat 値が必要なので次で対応付ける。

- `識別子候補` -> `2`
- `固有名候補` -> `5`
- `説明語候補` -> `1`
- `境界が曖昧な候補` -> `7`

### 3.2 慎重側へ倒す

単語だけで用途が定まりにくい語、小文字ハイフン語、規則交差語は `7` へ送る。

### 3.3 manual を実装準拠にする

manual のカテゴリ推定表を、旧 4 パターンの粗い説明から、

- 識別子系 -> `2`
- 混在大文字語 -> `5`
- 小文字空白フレーズ -> `1`
- 小文字ハイフン/単語単体/その他曖昧 -> `7`

へ寄せる。

## 4. 却下する案

- `project 固有語` を UI で自動推定する
  - 文脈依存が強すぎるため不採用
- UI と `veil-normalize.py` で完全共通関数化する
  - 今回は JS / Python 横断共通化まで広げない
