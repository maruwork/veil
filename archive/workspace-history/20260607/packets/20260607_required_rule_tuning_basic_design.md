# Required Rule Tuning Basic Design

## 1. Design Principle

- `必須` は fail-close なので、最小集合に寄せる
- exact current usage が弱い internal governance 語は `推奨` へ落とす
- current default profile は technical writing 向けなので、core technical writing / VEIL mainline の中核語だけを hard gate に残す

## 2. Rationale

### Keep Required

- `current state`
  - current canonical で sample / lint 例に直接出る
- `normalization`
  - VEIL core の中核概念
- `skill`
  - current canonical と実運用導線で高需要
- `task`
  - task close / task report で高需要

### Demote To Recommended

- `checkpoint`
  - common 骨格では強いが、返答前 hard gate 語としては広い
- `close verdict`
  - internal close operation 寄り
- `current surface`
  - governance 用語寄り
- `design`
  - 一般性が高い
- `design boundary`
  - internal planning 寄り
- `goal`
  - 一般性が高い
- `pending`
  - 統一価値はあるが hard gate までは不要
- `readback`
  - verification 運用語だが current hard gate からは外せる
- `source of truth`
  - 概念語としては有用だが hard gate までは不要
- `sync surface`
  - internal sync 運用語寄り
