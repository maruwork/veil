# VEIL 正規化判別補助 要件

Status: Draft
Date: 2026-06-07

## 1. 目的

`veil-normalize.py` に、候補語が識別子寄りか、固有名寄りか、説明語寄りかを早く見るための軽い判別補助を追加する。

## 2. 背景

- 頻度だけ見えても、変換対象外の識別子を先に候補へ送るとノイズが残る
- いまの判別ルールは文書にあるが、補助スクリプトの出力からは種類が見えない
- 高需要先行の前に「何の種類の語か」を早く切れると運用しやすい

## 3. 今回の範囲

- `veil-normalize.py` の出力へ軽い判別補助を追加する
- text / json 出力の両方に反映する
- README、設計書、capture skill に補足を入れる

## 4. 今回の範囲外

- 完全自動のカテゴリ確定
- `app.py` のカテゴリ自動推定変更
- `veil-lint.py` の変更
- `project 固有語` の自動判定

## 5. 機能要件

### 5.1 判別補助

少なくとも次の補助ラベルを返せること。

- `識別子候補`
- `固有名候補`
- `説明語候補`
- `境界が曖昧な候補`

### 5.2 根拠

補助ラベルには、少なくとも 1 つの簡単な理由を添えること。

例:

- パスや拡張子に見える
- `key=value` に見える
- 内部大文字に見える
- 一般的な小文字フレーズに見える
- 規則が交差して判定が揺れる

### 5.3 慎重側の判定

- 迷う時は `境界が曖昧な候補` 側へ寄せる
- `project 固有語` は自動決めしない
- 最終判断は user に残す

## 6. 非機能要件

- 軽量ヒューリスティックに留める
- 既存の頻度補助と競合しない
- 日本語説明を優先する

## 7. 完了条件

- `veil-normalize.py` の text 出力で補助ラベルと理由が見える
- `--json` 出力で `classification_hint` と `classification_reason` が取れる
- 文書側に「これは最終判定ではない」と明記される
