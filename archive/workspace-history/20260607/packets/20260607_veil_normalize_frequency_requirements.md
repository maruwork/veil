# VEIL 正規化頻度補助 要件

Status: Draft
Date: 2026-06-07

## 1. 目的

`veil-normalize.py` を、高需要語から段階導入する運用に使える補助スクリプトへ寄せる。

今回は、候補語の重複出現を保持し、正規化後の頻度と採用順の目安を返せるようにする。

## 2. 背景

- 現在の `veil-normalize.py` は候補語の重複を早い段階で消している
- そのため、どの語が頻出かを出力から読めない
- 段階導入では、少なくとも頻度の高い語から先に見る判断材料が必要

## 3. 今回の範囲

- `veil-normalize.py` が候補語の出現回数を保持する
- 正規化後クラスタ単位の頻度を返す
- 頻度に基づく簡単な採用順の目安を返す
- text / json 出力の両方に反映する
- README、設計書、capture skill に補足を入れる

## 4. 今回の範囲外

- 困り度の自動判定
- UI の頻度計算変更
- `veil-lint.py` のロジック変更
- `~/.veil/rules/` の実データ自動整理

## 5. 機能要件

### 5.1 出現回数保持

- 入力候補語の重複を消さずに読み取れること
- 同じ表記と表記揺れの両方について回数を持てること

### 5.2 正規化後頻度

- 正規化済みキーごとに合計出現回数を返すこと
- 各 variant ごとの回数も返せること

### 5.3 採用順の目安

- 頻度だけに基づく簡単な採用順の目安を返すこと
- 目安は「先に見る」「次に見る」「保留候補」程度の軽い補助に留めること
- 困り度の最終判断は user 側に残すこと

### 5.4 並び順

- 結果は、基本的に出現回数の多い順で返すこと
- 同頻度では normalized key 昇順など安定順序にすること

## 6. 非機能要件

- 既存の `existing-match` / `new-candidate` の意味は壊さない
- 競合警告の動作は維持する
- 日本語説明を優先する

## 7. 完了条件

- `veil-normalize.py` の text 出力で頻度と採用順目安が見える
- `--json` 出力で cluster 頻度と variant 頻度が取れる
- README / 設計書 / skill に「頻度を見て先に採用する」補足が入る
