# veil-capture 実行ゲート付き手順

## 0. この手順の目的
この手順は、会話からAIが使った英語・造語・略語を検出し、日本語に変換して `~/.veil/veil.db` canonical に記録し、`~/.veil/rules/` mirror を生成して `shared/runtime/veil-sync.py` まで完了するための実行手順である。  
確認だけで終わらせず、各段階で完了条件を満たした場合にのみ次へ進む。  
1つでも未達がある場合は、その場で停止し、未達理由を報告する。

この手順は、task close や会話区切りで通す VEIL の閉じ処理として使う。VEIL は `AI-assisted technical writing` 向け terminology guardrail であり、全語彙を一気に hard gate へ上げるための手順ではない。

**出力ルール**: ステップ 1〜7 の処理はユーザーへ出力しない。ユーザーへの出力はステップ 8 のテンプレートのみとする。

---

## 1. 解析対象の確定

### 入力
- 引数文字列、または現在の会話全体

### 実行
- 引数がある場合は、その文字列だけを解析対象にする
- 引数がない場合は、この会話全体を解析対象にする
- 会話全体を使う場合は、ユーザー発言とAI発言の両方を含める

### 禁止
- 解析対象を曖昧なまま進めること
- 会話全体対象なのにAI発言だけ、またはユーザー発言だけで処理すること

### 完了条件
- 解析対象が1つに確定している
- 会話全体対象の場合、ユーザー発言とAI発言の両方を含むことが確認できている

### 未達時の停止
- 解析対象が確定できない場合は停止し、その理由を報告する

---

## 2. 候補抽出

### 入力
- 1で確定した解析対象テキスト

### 実行
- 単語単位ではなく、意味が崩れない最短のまとまりで候補を拾う
- 単語より複合語を優先する
- 同じ並びで2回以上出る語だけを候補に残す
- 2回以上出るだけでなく、状態語、判断語、構造語、運用ラベルを優先する
- 一般動詞単体は原則として候補に送らない
- 一般動詞を扱うのは、複合語で意味が固まる場合か、同じ運用文脈で繰り返し出る場合だけにする
- 文中に既存の自然な日本語表現がある場合はそれを候補にする

### 禁止
- 1回しか出ていない語を候補にすること
- 複合語で取れるのに単語へ分解すること
- 意味単位ではなく見た目だけで区切ること
- 一般動詞単体を安易に候補へ送ること

### 完了条件
- 候補語の一覧が作られている
- 各候補が2回以上出現している
- 各候補が意味単位または複合語として妥当である
- 一般動詞単体が機械的に混入していない

### 未達時の停止
- 候補抽出基準を満たせない場合は停止し、その理由を報告する

---

## 3. 除外判定

### 入力
- 2で作成した候補語一覧

### 実行
- 固有名詞を除外する
- 製品名、サービス名、ツール名、組織名、正式機能名を除外する
- バッククォートで囲まれた語を除外する
- ダブルクォートで囲まれた語を除外する
- `key=value` 形式を除外する
- 正式名、識別子、仕様名、コマンド名を一般語として登録しない
- 開発者が英語のまま使うことが一般的な業界標準技術語は除外する
  - 判断基準: 日本語のドキュメントや口頭でも英語表記が出てくる語
  - 迷う場合は step 5 のカタカナルールで処理する
- 固有名詞か一般語か判断できない場合は除外（保留候補へ移す）

### 禁止
- 固有名詞か迷う語をそのまま通すこと
- コード識別子やパスを一般語として登録すること
- 除外判定を飛ばして次へ進むこと

### 完了条件
- 除外対象が候補一覧から外れている
- 除外した理由を説明できる
- 迷う語は保留候補に移されている

### 未達時の停止
- 除外判定が完了していない場合は停止し、その理由を報告する

---

## 4. 正規化

### 入力
- 3を通過した候補語一覧

### 実行
- 大文字小文字を小文字に統一する
- 単数形と複数形を単数形に寄せる
- ハイフンとアンダースコアをスペースに統一する
- 正規化後の形で同一語判定を行う

### 禁止
- 生の表記ゆれのまま重複判定すること
- 正規化前後で別語として数えること

### 完了条件
- 候補語が正規化済みである
- 重複語が統合されている

### 未達時の停止
- 正規化が終わっていない場合は停止し、その理由を報告する
- 必要な正規化なのに variant collapse を確認していない場合は停止し、その理由を報告する

### 補助ツール
- 候補語の揺れが明らかな場合は `python shared/runtime/veil-normalize.py --stdin` 相当で正規化と統合候補を確認する
- `既存一致:` グループに出た語は既存ルールとの照合済みとして扱う
- `新規候補:` グループの語は `x{N}` の N が大きい語から採用検討に回す
- single-word の一般動詞と軽い活用形は、頻度だけで説明語へ上げず保守側のまま扱う
- repeated single-word 一般語も、2回出現しただけでは自動で説明語候補へ上げない
- single-occurrence の lowercase phrase も、1回だけでは自動で説明語候補へ上げない
- repeated lowercase phrase も、2回出現しただけでは自動で説明語候補へ上げない
- `project 固有語` の最終判断は補助出力に任せず user 側で行う
- 補助出力は authority ではなく、最終判断は SQLite canonical または current mirror を読んで行う

---

## 5. 判別・優先度付け・表記候補の決定

### 入力
- 4で正規化した候補語一覧

### 実行
- まず各候補を、採用する、スキップする、のどちらかへ判別する
- 識別子は登録対象へ送らない
- 固有名は、表記統一の必要がある場合だけ採用する
- 説明語と定義語を採用する
- 訳語が決まらない語はスキップする（次に出現したときに改めて判断する）
- `close / closed / closing / updates` のような一般動詞系 single-word は複数回出てもスキップを優先する
- normalize の `新規候補:` グループは `x{N}` の N が大きい語から先に確認する
- normalize の `既存一致:` グループは照合済みとして扱い、preferred を確認する
- 採用候補は次の順で優先度を付ける
  - 頻出で困り度が高い語
  - VEIL運用そのものに効く基幹語
  - プロジェクト固有語
  - 低頻度語
- 今回は上位から必要分だけ採用し、残りはスキップする
- 業界標準技術語はカタカナを優先する（例: canonical → カノニカル、mirror → ミラー）
- 日本語訳を使うのは、一般的な日本語表現が定着している語に限る（例: false positive → 誤検知、governance → ガバナンス）
- 英語維持は識別子、正式名、仕様名、コマンド名だけに限定する
- 1概念1表現で候補を決める

### 禁止
- 判別前に訳語決定へ進むこと
- 候補を一気に全部採用すること
- 新語を安易に作ること
- 同じ概念に複数の候補1を作ること
- 迷う語を無理に登録すること

### 完了条件
- 各候補が採用またはスキップに判別済みである
- 各採用語に候補1・候補2が決まっている（候補3は任意）

### 未達時の停止
- 判別が終わっていない場合は停止し、その理由を報告する
- 採用優先度を付けていない場合は停止し、その理由を報告する
- 採用語に候補2が決まっていない場合は停止し、決定してから進む

---

## 6. 既存ルール照合

### 入力
- 5で確定した候補語一覧
- `~/.veil/veil.db`

### 実行
- 候補が1件以上ある場合、`python shared/tools/veil-db.py readback --db ~/.veil/veil.db --json` 相当または mirror 読取で既存 entry を必ず先に確認する
- canonical DB が存在しない current fallback の場合だけ mirror を既存 entry source として扱う
- 既存エントリと元語が一致する場合はスキップする
- 重複判定は大文字小文字を区別せずに行う
- 必要なら `shared/runtime/veil-normalize.py` の既存一致提示を使って統合候補を確認する

### 禁止
- canonical DB または mirror を読まずに書き込み内容を確定すること
- 既存エントリを確認せずに追加すること
- 重複判定を省略すること

### 完了条件
- 追加対象と既存重複スキップ対象が分かれている
- canonical DB の事前照合が終わっている

### 未達時の停止
- 既存ルール照合が終わっていない場合は停止し、その理由を報告する

---

## 7. 記録内容の決定

### 入力
- 6で確定した新規追加対象一覧

### 実行
- canonical への記録単位は 1 rule ごととする
- mirror 振分は `shared/tools/veil-db.py export-mirror` が担うので、capture 側で file path を正本として扱わない
- preferred が決まらない語は書き込まずスキップする

### 禁止
- section 未確定のまま書くこと

### 完了条件
- すべての新規語に preferred が割り当てられている

### 未達時の停止
- preferred が未確定の語がある場合は停止し、その理由を報告する

---

## 8. 選択肢の提示

### 入力
- 7で確定した追加対象一覧

### 実行
以下のテンプレートをそのまま使う。テンプレート外の文章は出力しない。

```
- {term}（現状） → {候補1}（候補1）| {候補2}（候補2）| {候補3}（候補3）

現状、または候補を選択してください。
```

### テンプレートのルール
- 採用語ごとに1行出力する
- 候補2は必ず出力する
- 候補3 がない場合はその `|` 以降を省く
- 採用語が0件の場合は `採用語なし` の1行のみ出力して終了する

### 禁止
- テンプレート外の文章を出力すること
- 採用語の形式を崩すこと
- ユーザー応答を待たずに次へ進むこと

### 完了条件
- テンプレート形式で出力されている
- ユーザーの選択が確認できている

### 未達時の停止
- ユーザーの選択が確認できない場合は停止し、選択を求める

---

## 9. canonical 記録と mirror 生成

### 入力
- 8でユーザーが選択した語と表記
- `~/.veil/veil.db`

### 実行
- 各語についてユーザーが選択した表記を preferred として `python shared/tools/veil-db.py upsert-rule` 相当で canonical DB へ追加更新する
- 候補2、候補3も同時に渡す
- 記録後に `python shared/tools/veil-db.py export-mirror` 相当で markdown mirror を再生成する

### 禁止
- ユーザーの選択を無視して候補1を書き込むこと
- canonical を飛ばして mirror 直書きすること
- mirror 再生成を省略すること

### 完了条件
- canonical DB への追加更新が確認できている
- mirror 再生成が確認できている

### 未達時の停止
- DB 記録または mirror 再生成が確認できない場合は停止し、その理由を報告する

---

## 10. veil-sync 実行

### 入力
- 更新済み `~/.veil/veil.db`
- 更新済み `~/.veil/rules/*.md`
- `~/.veil/config.json`

### 実行
- `~/.veil/config.json` を読む
- `sync_script` キーを確認する
- `sync_script` があれば `shared/runtime/veil-sync.py` を実行する

### 禁止
- `config.json` を見ずに同期先を推測すること
- 書き込み後に同期せず完了扱いにすること

### 完了条件
- `shared/runtime/veil-sync.py` の実行結果が確認できている

### 未達時の停止
- `config.json` がない、または `sync_script` がない場合は停止し、その理由を報告する
- `shared/runtime/veil-sync.py` 実行結果が確認できない場合は停止し、その理由を報告する

---

## 11. 最終ゲート

### 次へ進んでよい条件
- [ ] 1から10まで、各段階の完了条件をすべて満たした
- [ ] 未達時停止に該当する項目が残っていない
- [ ] 候補抽出、ユーザー選択、DB 書き込み、同期の全段階が終わっている

### 進んではいけない条件
- [ ] 1つでも未確認の段階がある
- [ ] 1つでも省略した段階がある
- [ ] ユーザー選択前に DB 書き込みをしている
- [ ] 既存照合前に登録を確定している
- [ ] 同期前に完了扱いしている

### 終了判定
- 上の「次へ進んでよい条件」をすべて満たした場合のみ完了
- それ以外は未完了として停止し、未達理由を報告する
