# /veil-capture — チャットからVEILルールを抽出

会話テキストを解析し、AIが使った英語・造語・略語を検出して日本語に変換し、`~/.veil/rules/` に書き込む。

## 手順

### 1. 解析対象テキストの決定

- `args` が指定されている場合 → その文字列を解析対象とする
- 指定がない場合 → この会話全体を解析対象とする（ユーザー発言・AI発言の両方を含む）

### 2. 候補の絞り込み（この順番で実行する）

#### ステップ1: 意味単位で拾う

単語単位ではなく、**意味が崩れない最短のまとまり**を1用語とする。

- `current state` → 1用語
- `current` 単独 → `current state` が存在するなら保留
- `common asset promotion` → 1用語候補

#### ステップ2: 複合語を単語より優先する

2語以上が固定の並びで繰り返し出るなら、その複合語を先に候補にする。
単語単独の登録は、その複合語で処理できない時だけ。

複合語の判断基準：
- 分けると意味が変わる（`current state` → `state` だけでは別物）
- 文書内で同じ並びで2回以上出現している
- 片方だけ訳すと不自然になる

#### ステップ3: 反復していない語は捨てる

**1回しか出ない語は登録しない。** 同じ意味・同じ並びで複数回出ることが登録の最低条件。

#### ステップ4: 固有名詞を除外する

製品名・サービス名・ツール名・組織名・正式機能名は除外。
役割語・状態語・判断語・構造語だけを候補に残す。

#### ステップ5: 既存表現と照合する

文書内にすでに日本語表現がある場合は、それを使う。新しい言い換えは作らない。

候補が1件以上ある場合、**必ず** 対象の `~/.veil/rules/{先頭文字}.md` を読み込んでから照合する。ファイルが存在しない場合は既存エントリなしとして扱う。元語が一致する場合はスキップ（大文字小文字を区別しない）。

#### ステップ6: 表記を正規化してから同一語判定する

- 大文字小文字 → 小文字統一
- 単数・複数形 → 単数形に統一（`states` → `state`）
- ハイフン・アンダースコア → スペースに統一（`current-state` = `current state`）

#### ステップ7: 日本語 / カタカナ / 英語維持を基準で決める

| 表記 | 条件 |
|------|------|
| 日本語 | 状態・判断・行為・構造を表す語。日本語訳しても意味が落ちない |
| カタカナ | 業界で定着しており、日本語訳が不自然な語 |
| 英語維持 | 識別子・正式名・仕様名・コマンド名のみ |

**新語は原則禁止。** 既存語で言えない時だけ保留にする。
**1概念1表現。** 同じ意味に複数の言い方を作らない。

#### ステップ8: 迷うものは保留する

以下のいずれかに当てはまる場合は登録せず、保留候補として完了報告に記載する：
- 1語で取るか複合語で取るか迷う
- 日本語かカタカナか迷う
- 固有名詞か一般概念か迷う

#### 除外する文脈

- バッククォート（`` ` ``）で囲まれた語 → コード識別子・ファイルパスとして除外
- ダブルクォートで囲まれた語 → 固有値として除外
- `key=value` パターン → 設定値として除外

### 3. ファイルへ書き込む

#### ファイルテンプレート

```markdown
# {letter}

- {term} → {訳語}
```

- ヘッダー `# {letter}` は先頭文字（小文字）
- エントリは `- {元語} → {推奨表記}` の形式で1行1エントリ
- エントリは元語のアルファベット昇順に並べる

#### 書き込み先

| 条件 | ファイル |
|------|---------|
| a〜z で始まる語 | `~/.veil/rules/{先頭文字}.md` |
| 数字・記号で始まる語 | `~/.veil/rules/special.md` |

先頭文字は小文字に統一。

#### 書き込み手順

1. 対象ファイルを読み込む（存在しない場合はテンプレートから新規作成）
2. 既存エントリと大文字小文字を区別せずに元語を比較し、一致する場合はスキップ
3. 新規エントリを追加し、全エントリを元語の昇順（case-insensitive）で並べ替えて上書き保存

### 4. veil-sync を実行

ファイル書き込み後、`veil-sync.py` を実行して同期先ファイルに即時反映する。

`~/.veil/config.json` の `sync_script` キーからパスを取得する。

```bash
# bash / zsh (macOS / Linux)
python "$(python3 -c "import json; print(json.load(open('$HOME/.veil/config.json'))['sync_script'])")"
```

```powershell
# PowerShell (Windows)
python (python -c "import json,os; print(json.load(open(os.path.expanduser('~/.veil/config.json')))['sync_script'])")
```

`config.json` が存在しない、または `sync_script` キーがない場合はユーザーに場所を確認する。
VEILサーバーが起動していない場合は同期をスキップし、その旨を報告する。

### 5. 完了報告

抽出した語を以下の形式で出力する。

```
common asset → 共通資産（候補1）、コモンアセット（候補2）、共通アセット（候補3）
current state → 今の状態（候補1）、カレントステート（候補2）
validator → バリデータ（候補1）、検査（候補2）
```

末尾に必ず以下を固定で出力する：

> 修正指示がない場合は、候補1が `~/.veil/rules/` に格納されます。

ユーザーが対案を提示した場合は、その語を候補1として書き込む。

保留がある場合は末尾に追記する：

```
保留: current（current state の一部として出現、単独登録は不要と判断）
```

## 補助確認（最後の確認にのみ使う）

以下は主観に寄りやすいため、中核ルールではなく最終確認として使う：
- 初見の人が1回で分かるか
- 1語で済むなら1語
- カタカナが自然かどうか
