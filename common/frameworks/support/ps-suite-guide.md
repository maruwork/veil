# PS Suite Guide

PS Suite is not a core rule of `pj-template`.
Treat it as a support toolkit for reading, organizing, discovering, and reflecting.

## Use When

- information is scattered and you want to align the viewpoints first
- you want to separate divergence from convergence
- you want to switch the view across business, design, and implementation
- you want a reflection viewpoint after the work

## How To Use It

1. read `../core/progression-rule.md` and the project's current rule first
2. use PS Suite only as a support viewpoint
3. do not substitute canonical authority with PS Suite output
4. promote only reusable viewpoints into policies or frameworks

## What It Must Not Do

- decide the current canonical source
- stand in for owner judgment
- make the final ruling on project-specific rules
- replace branch decisions
